## 瑞克的核心動力模式（2026-05-23 自我盤點）

> 詳見 `daily/2026-05-23.md`。莫蒂在做重大決策建議時要記得這份脈絡。

- **童年線索 → 現在的線**：畫畫 → 設計 → 視覺品牌（一條沒斷）
- **心流時刻**：設計、創造東西的時候
- **耗能時刻**：處理客戶要求的時候
- **核心張力**：ZOXE 的商業模式 =「處理客戶要求 → 產出設計」，跟他的心流模式有結構性衝突
- **Rick Z. 個人品牌**：不只是 ZOXE 引流工具，可能是他「真正想要」的那條線本身
- **室內設計師客群**：邏輯成立（有預算 + 有 know-how），但屬於「正確答案」而非「真心想要」
- **潛水**：喜歡 ≠ 想當教練。目前定位成「Rick Z. 的故事背景而非服務」是對的

### 決策提醒清單

當瑞克問以下問題時，調用上面脈絡：
- 「ZOXE 要不要做 XX 服務？」→ 會增加「處理客戶要求」比重嗎？
- 「Rick Z. 要不要做 XX 內容？」→ 別把它當引流工具，它可能是主場
- 「要不要接 XX 案子？」→ 客戶有預算 ≠ 這條路適合長期投入
- 「要不要往 XX 方向轉？」→ 能讓「創造 > 服務」的比例提高嗎？

## Feedback（AI 學到的原則）

### 寫作風格 7 條（2026-05-23 從一篇 Threads 心得文修改學到）

幫瑞克寫貼文、長文時要遵守：

1. **思考感 > 情緒反應**
   - ❌「超扎心」「呆住」「整個人 XX 了」
   - ✅「感受很深」「陷入思考」「有點難堪」
2. **保留尊嚴的自嘲**
   - ❌「我不夠拚而已啦」「我擺爛」這種貶低自己
   - ✅「我以前以為時間到就會有辦法」這種「正在覺醒的人」語氣
3. **不用粗口、不用爆點 emoji**
   - ❌「靠」「幹」「爽爽」「😅」「😆」太頻繁
   - ✅ 中性、克制、像在自己日記裡寫東西
4. **個人故事要有感官細節**
   - ❌「我從小喜歡設計」（抽象）
   - ✅「盯著圖看很久、觀察它、觸摸它、甚至放到嘴裡感受硬度」（具體）
5. **直接稱呼專業，不要繞圈**
   - ❌「視覺」「美學」「創作」
   - ✅「設計」「網站」「個人品牌」
6. **開頭可以下標題式鉤子**
   - 例如：「如何找到人生目標？」放最上面，再進入內文
7. **結尾收乾淨**
   - 附連結就附連結，不要再追加「我看完有點想躺下發呆」這種自嘲段

### 標點符號偏好
- **用全形標點**（逗號 `，`、冒號 `：`、問號 `？`），不要半形
- 段落間留空行，方便閱讀
